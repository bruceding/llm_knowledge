# 前端国际化（i18n）设计文档

日期: 2026-04-25

## 需求概述

在 worktree 中开发前端国际化功能，支持 UI 界面中英文切换。

- **UI 文字**: 需要中英文切换（导航、按钮、标签等）
- **文档内容**: 保持原始语言显示（英文文档显示英文，中文文档显示中文）
- **语言偏好**: 存储到数据库，记住用户选择
- **默认语言**: 首次访问默认英文

## 技术方案

使用 **react-i18next** 库实现国际化。

## 架构设计

```
frontend/src/
├── i18n/
│   ├── index.ts          # i18next 配置初始化
│   ├── locales/
│   │   ├── en.json       # 英文翻译
│   │   └── zh.json       # 中文翻译
├── contexts/
│   └── SettingsContext.tsx  # 用户设置状态管理
├── pages/
│   └── SettingsPage.tsx  # 设置页面
└── components/
    └── LanguageSelector.tsx  # 语言选择组件
```

## 数据模型

### 后端 SQLite

```sql
user_settings
├─ id (INTEGER PRIMARY KEY)
├─ language (TEXT) — 'en' | 'zh'
├─ created_at (DATETIME)
├─ updated_at (DATETIME)
```

### API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/settings | 获取用户设置 |
| PUT | /api/settings | 更新设置（包含语言字段） |

## UI 改动

1. **Sidebar**: 底部添加 Settings 入口（齿轮图标）
2. **SettingsPage**: 包含语言下拉选择（English / 中文）
3. **现有组件**: 使用 `useTranslation()` 替换硬编码文字

## 翻译范围

需要翻译:
- Sidebar 导航标签（Inbox、Documents、Tags、Wiki 等）
- 页面标题、按钮文字
- 表单标签、提示信息
- 错误提示、状态文字

不需要翻译:
- 文档标题、内容
- Wiki 页面 markdown 内容
